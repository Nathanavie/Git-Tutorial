Staging

Create File

Create the file that will be used within your project.

git add .

This will add all modified and new files into the staging area.

git commit -m "changed feature-x in file-x"

This will add a message to your staged file.

Adding and Committing files

The next thing you will need have an understand of is how to add and commit files

This means that you propose a change by adding it to the Index tree from the Working Directory tree, to do this use the command

git add 'filename' to add a specific file git add . to add all changes

This is the first step in a basic git workflow, the next step is to commit these changes to the Head tree. To do this use

git commit -m "commit message"

A good commit message is no longer than 72 characters long, but includes all changes that are being committed.
