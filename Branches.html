When developing different features within a project, it is important to ensure that branches are used correctly.

Branches are used to devlop different features that are isolated from each other, with the master branch being the default branch that is created when the repository is created. The main idea with branches is to use other branches to develop features and then merge all the branches together to the master branch for a complete project.

To create a new branch, use the code

git checkout -b feature_x

To switch to another branch that is already created, such as master

git checkout master

To delete a branch

git branch -d feature_x

It is important to understand that a branch is not accessible by others unless the branch is pushed to the remote repository with

git push origin 'branch'

Merging Branches

If the development on a branch is complete and it is time to merge the branch the master branch, use the code

git merge "branch"

This merges whichever branch you specified into your active branch, so ensure that your active branch is the master branch if you are doing a final merge!

Git will usually try to auto merge changes when doing this, however sometimes it is not possible and there will be comflicts found. To fix this you will have to merge these specific files manually by editing the files that are shown in git.

After you have fixed any conflicts, add these files as merged by using

git add "filename"

If you are unsure on the differences between two branches, you can preview each branch by using

git diff "source-branch" "target-branch"

This will allow you to understand what differences there are between two branches in your repository.
